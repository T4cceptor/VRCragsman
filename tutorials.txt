Voraussetzungen:
Visual Studio 2012 Express/Professional/Ultimate (https://www.microsoft.com/de-de/download/details.aspx?id=34673)
CMake 3.2 oder neuer (http://www.cmake.org/files/v3.2/cmake-3.2.2-win32-x86.exe)
CMake bei der Installation für alle Benutzer den Pfad setzen lassen.

Das Archiv "libraries.zip" bitte ins Stammverzeichnis von C: entpacken.
C:\
 +- libraries
     +- install
         +- vs11_32
             +- boost
             +- freeglut
             ...
             +- vrpn


Das Archiv "tutorials.zip" in ein beliebiges persönliches Verzeichnis entpacken.
Dort die Datei "vs11_32_osg_invrs.cmake" auf "run_cmake.bat" ziehen (Drag'n'drop). (Cmake erstellt nun unter "build_vs11_32_osg_invrs" die Projekt-Dateien für Visual Studio. Danach wird ein Shortcut "start_vs_vs11_32_osg_invrs" erstellt.)
Mit dem erstellten Shortcut (Verknüpfung auf "build_vs11_32_osg_invrs/tutorials.sln") starten sie die Entwicklungsumgebung mit den einzelnen Projekten (Release-Version).
(Alternativ ist vs11_32_osg-only.cmake vorhanden. Hiermit kann entsprechend verfahren werden. Hier sind Debug- und Release-Versionen vorhanden, allerdings nur für reine OpenSG-Projekte ohne InVRs.)

Den csm-Renderserver können sie bei Bedarf mit "start_csm" (Verknüpfung auf "C:\libraries\install\vs11_32\invrs\bin\start_csm.bat") starten.
Die lokale Windows-Firewall brauchen sie nicht zu öffnen. Falls ein Dialog-Fenster auftaucht, einfach "Abbrechen" betätigen.
